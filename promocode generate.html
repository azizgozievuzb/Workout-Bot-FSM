FSM: Onboarding & Code Architecture

Глобальные правила генерации кодов

R Код Ответственного (Responsible)
P Код Игрока (Player)
Мин. длина: 8 символов
ROOT ENTRY

1. Регистрация Админа

Ввод hard_code_admin

Выдает сразу две роли: Admin + Responsible

ЭКРАН: ACTION

2a. Первая пара (Root)

Авто-генерация кода:
P_UNLIM_88X

Код исчезнет после 1 использования.

Тестовый Аккаунт

Вводит код → Становится Игроком.

Итог: Создана уникальная пара Root-Admin с Игроком (Безлимит).

ЭКРАН: CUBE (Генератор)

2b. Фабрика кодов Админа

Генерация новых кодов вручную.

Путь А: Player

ТОЛЬКО для продления!

Указанная длительность зашивается в этот код (напр. P_RENEW_30D). Как только он активируется, эта длительность начинает отсчитываться.

Путь Б: Responsible

Админ генерирует код для нового Ответственного (напр. R_COACH_99Y) и задает дату (например, 30 дней).

НОВЫЙ ОТВЕТСТВЕННЫЙ

Его Action Экран

Авто-генерация кода для Игрока (напр. P_AUTO_55Z).

Магия наследования: Этот авто-код автоматически получит длительность в 30 дней (заложенную Админом шагом ранее).
UI ЗАГЛУШКИ (В разработке):
[Купить код для нового игрока (Stars)]
[Продлить срок игроку (Crypto)]
Текущий реальный флоу продления:
1. Ответственный просит код у Админа (Путь А).
2. Передает код своему Игроку.
3. Игрок вводит его сам и продлевает доступ.
