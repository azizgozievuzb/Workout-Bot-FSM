## 🎬 Демо-видео упражнений — naming convention для метаданных

**Контекст:** сейчас демо-видео в `frontend/public/demos/<key>.mp4` имеют только ключ упражнения в имени. Метаданные (position, muscles, hint) хранятся отдельно в `backend/core/workout_config.py`. Когда юзер будет снимать собственные демо — нужно зашифровать ВСЁ в имя файла, чтобы:
- проще было распознавать видео без открытия конфига
- метаданные не «терялись» при перезаливке/перенарезке
- на frontend можно было бы показывать положение камеры («Встаньте боком», «Повернитесь спиной») АВТОМАТИЧЕСКИ из имени файла

**Предлагаемый формат имени файла:**

```
<exercise-key>__<difficulty>__<goal>__<position>__<camera-angle>.mp4
```

Где:
- `exercise-key` — slug упражнения (`squats`, `pushups`, `plank`)
- `difficulty` — `easy` | `medium` | `hard`
- `goal` — `endurance` | `strength` | `cutting` | `bulking` (выносливость / сила / похудение / набор)
- `position` — `standing` | `sitting` | `lying-back` | `lying-front` | `kneeling` | `plank-pos`
- `camera-angle` — `front` | `side-left` | `side-right` | `back` | `top-down`

**Примеры:**
```
squats__medium__endurance__standing__front.mp4
plank__hard__strength__plank-pos__side-left.mp4
crunches__easy__cutting__lying-back__front.mp4
```

**Что делать в коде когда дойдём:**
1. Backend: парсер имени файла → извлекает метаданные при загрузке. Поля `Exercise` обогащаются: `difficulty`, `goal`, `position`, `camera_angle`.
2. Frontend: на rest-экране показывать «Встаньте боком к камере / лицом к камере / спиной к камере» по `camera_angle`.
3. Frontend: фильтрация в Shop по `goal` и `difficulty` (для tier-based доступа).
4. Backend: при отсутствии метаданных в имени — фолбэк на дефолты + warning в логи.

**Когда делать:** когда юзер будет снимать собственные демо (после успешного прохождения 7.3-7.4 и переноса на полноценный продакшен).

**Связанные задачи:** content debt по замене 16 текущих демо (нарезанных Antigravity-агентом, не точно соответствующих названиям).

## 🟦 Фаза 8 — Workout Types & Drops Economy (DESIGN DOC)

> Полностью спроектировано в Session 36 (интервью). Реализация — отдельная фаза после 7.4 (Stars payments).

### Контекст
- **Звёзды переименованы → Капли (Drops 💧)**, чтобы не пересекаться с Telegram Stars (реальная валюта).
- Новая формула расчёта в **Session 36** (см. ниже, базовая часть закрывается в 7.3).
- **Идея фазы 8:** ввести **2 типа тренировок** + механика покупок light-режима + расширенная экономика капель.

### 8.1 Типы тренировок

**Main (heavy) — основная тренировка** (текущий формат):
- 16 упражнений × ~26 минут
- Максимум за сессию: **50 капель**
- Формула: `50 × quality × (done/16)^0.65 × streak_mult`

**Light (лёгкая) — облегчённая тренировка** (новая):
- 4 упражнения × ~20 минут (3 мин работа + 1 мин отдых × 4 + разминка/заминка)
- Максимум за сессию: **30 капель**
- Формула: `30 × quality × (done/4)^0.65 × streak_mult`
- Контент: ходьба на месте, подъём коленей, пятки к ягодицам, маршировка
- ⚠️ **Не позиционировать как «тренировка»** — называть «ритуал» / «активность» / «зарядка»
- Light-режим **НЕ открыт по умолчанию**, нужно купить за капли

### 8.2 Расписание

- **Жёсткое: 3 main-дня в неделю**, выбираются на онбординге (опрос «какие 3 дня?»).
- Можно выбирать 3 подряд дня, но с warning «для восстановления распределите с промежутками».
- После unlock light: 7-дневный график (3 main + 4 light).
- Все weekly-ресеты в **воскресенье 23:59 → понедельник 00:00 по локальной TZ юзера**.
- Смена графика — только в конце завершённой недели + дисклеймер. Платно за капли (TBD).
- **14-дневный grace period** на любые решения (онбординг → бесплатные смены первые 2 недели).

### 8.3 Streak-логика

**До unlock light:**
- Streak считают **только main-дни**.
- Пропустил main → streak ломается.

**После unlock light:**
- Streak считает **любой день** (main И light).
- Пропуск любого дня → streak ломается.

**Активация нового режима:** со следующего понедельника после unlock (вариант A1).
**Сломанный streak:** восстанавливается через `rest_days_remaining` (3 в месяц). Платный restore — в BACKLOG (квадратичная формула: `cost = 10 × days²`, кап разумный).

### 8.4 Light unlock — экономика отмены

- Покупка light = **навсегда** (one-time, не подписка).
- Можно отменить → следующая покупка стоит **×3** (вариант F-balanced).
- Сброс множителя через **6 месяцев** без отмены.
- Paused light — TBD (можно ставить на паузу неделю-две без сброса множителя).

### 8.5 Уведомления

- **2 в день максимум**: утреннее (день тренировки) + вечернее (если тренировки нет).
- **Inline-кнопки** ведут прямо в нужное состояние Mini App.
- **Время утреннего напоминания** выбирает юзер в настройках (опция B).
- **Snooze**: 3 раза в год бесплатно. Сверх — за капли.
- 2 snooze подряд → диалог «нужна помощь? Responsible видит».
- Подавление в `22:00–09:00` локального времени.

**Сообщения:**
```
🏋 Сегодня твой день тренировки!
[💪 Начать тренировку]   [⏰ Напомнить вечером]   [😴 Snooze]
```
```
⚠️ Сегодня ещё не тренировался. Стрик в опасности!
[💪 Начать сейчас]   [😴 Пропускаю (теряю стрик)]
```

### 8.6 Two-mode user (Player + Responsible)

Уже реализовано в коде через `primary_role + activeRoleView` (см. `frontend/src/stores/authStore.ts`).

**Расширения для фазы 8:**
- **2 столбца капель в `player_stats`/`users`:**
  - `xp_balance` — личные капли (заработанные тренировками).
  - `gift_balance` — подарочные капли (купленные за деньги для дарения).
- В режиме Responsible — **не виден** свой `xp_balance`.
- При переключении в Responsible — закончилась активная тренировка? Если не закончилась → **проваленная**.

### 8.7 Responsible — функции

- Дарит капли (купленные через codes/Stars/крипту, не свои заработанные).
- Размещает товары на **«своей полке» в магазине Player'а**.
- Не видит баланс капель Player'а.
- Не видит магазин Player'а целиком — только **свою полку**.
- Видит **прогресс Player** = XP (не капли). История за месяц + накопительная аналитика.
- Лимит подопечных: **standard=1, premium=2, elite=3** (уже в коде, `TIER_PLAYER_LIMITS`).

### 8.8 Магазин Player

**2 полки:**
- **Полка от Responsible** — кастомные предметы, расположенные наставником. Лимит слотов (TBD).
- **Полка от приложения** — стандартные товары/бусты.

**UX:**
- Player может нажать на товар от Responsible → «Мне это неинтересно» → товар скрывается, Responsible получает уведомление.
- Маркировка: «🎁 от твоего наставника».

### 8.9 Pair-only philosophy

- Player **не существует** без Responsible. Соло-режим невозможен.
- При запуске бот предлагает: «найди наставника» / «стань наставником».
- При исчезновении Responsible (не платит) — Player **теряет доступ** через 3 дня grace.
- Расторжение связи **невозможно**. Только через техподдержку (admin вручную в БД).
- ⚠️ **Safety button** — high priority перед public release. Сейчас отложено.

### 8.10 Дисклеймер-тесты

Любое irreversible action → дисклеймер + тест из 3 вопросов с правильными ответами.
**Хранить в коде** (хардкод). При неправильном ответе — попробовать снова.

**Применять к:**
- Unlock light
- Cancel light
- Смена графика (после grace)
- Привязка к Responsible-коду
- Удаление аккаунта
- Любые покупки за капли > 100

### 8.11 Drops decay

- Истечение через **6 месяцев бездействия** (= не сделал ни одной тренировки).
- Warning через Telegram-бота **за 14 дней** до сгорания.
- Открытие app не считается активностью.

### 8.12 Timezone

- Хранится в `users.timezone` (новое поле, миграция).
- Автоопределение через JS: `Intl.DateTimeFormat().resolvedOptions().timeZone` на онбординге + при каждом запуске.
- При расхождении — диалог «Похоже ты в LA. Обновить?» (вариант c).
- **H-soft mode**: смена TZ ничего не ломает, тренировки текущего «логического дня» сохраняются.
