# Архитектура Telegram Бот (Aiogram 3, PostgreSQL, Redis)

## Структура проекта

* `app/` — Главная папка с исходным кодом бота.
    * `core/` — Глобальные настройки. Здесь мы считываем конфигурации из файла `.env` (например, через `pydantic-settings`).
    * `database/` — Все, что связано с PostgreSQL.
        * `models/` — Описание таблиц базы данных в виде классов (SQLAlchemy / Tortoise ORM).
    * `handlers/` — Роутеры и обработчики (те самые `echo`, `/start`, нажатия на инлайн-кнопки).
    * `keyboards/` — Функции-построители кнопок (Inline и Reply).
    * `middlewares/` — Промежуточное ПО (мидлвари). Перехватчики сообщений (чтобы вставить сессию БД или отфильтровать спам).
    * `services/` — Бизнес-логика, не связанная напрямую с интерфейсом Telegram (работа с API, сложные вычисления).
    * `states/` — Группы состояний FSM (для работы с Redis: регистрация, ввод данных).
* `tests/` — Папка для автотестов.
* `.env` — Секретные ключи (токены, пароли баз). Игнорируется Git'ом!
* `main.py` — Точка входа, запуск бота.
* `requirements.txt` — Список зависимостей Python.
